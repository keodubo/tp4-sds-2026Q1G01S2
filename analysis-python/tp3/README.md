# TP3 Animation

Postprocessing-only animation for TP3 System 1 snapshot outputs.

The script reads `snapshot.txt` generated by the TP3 event-driven motor. It
does not run or modify the simulation. Each animation frame is a recorded event
snapshot, so the video should be described as event-by-event playback, not as a
fixed-time-step simulation.

## Usage

From the repository root:

```bash
PYTHONPATH=SdS_TP3_2026Q1G01CS2_Codigo/src \
  python3 -m tp3_sds system1 run \
  --config SdS_TP3_2026Q1G01CS2_Codigo/configs/system1.animation-smoke.toml

python3 analysis-python/tp3/animate_tp3.py \
  --input outputs/tp3-animation-smoke \
  --output outputs/tp3-animation-smoke/animation.mp4 \
  --fps 12 \
  --frame-stride 1 \
  --overwrite
```

## Animate `local_run_outputs.sh` Results

After the full local script finishes, TP3 runs live under:

```text
outputs/tp3-reference/tp3-final-grid/raw/N_<N>/r_<seed-index>/
```

Animate one run by pointing `--input` at the run directory:

```bash
python3 analysis-python/tp3/animate_tp3.py \
  --input outputs/tp3-reference/tp3-final-grid/raw/N_100/r_00 \
  --output outputs/tp3-reference/tp3-final-grid/raw/N_100/r_00/animation-preview.mp4 \
  --fps 6 \
  --frame-stride 1 \
  --overwrite
```

Generate one preview per TP3 run:

```bash
python3 analysis-python/tp3/animate_tp3.py \
  --batch-root outputs/tp3-reference/tp3-final-grid/raw \
  --output-name animation-preview.mp4 \
  --fps 6 \
  --frame-stride 1 \
  --overwrite
```

For a quick smoke preview, add `--max-frames 120`. The animations are generated
next to each ignored raw output, so they are not committed.

Particles use the RGB colors already written by the motor:

- green: fresh
- purple: used

The overlay shows the event id, physical time, delta time since the previous
rendered event snapshot, and used-particle count.
